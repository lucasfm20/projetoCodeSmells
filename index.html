Data Clumps (Aglomerado de Dados)

Equipe: Lucas Fortunato Martins e Kauã Machado Grathwohl

O que é Data Clumps?

Data Clumps ocorre quando certos grupos de dados aparecem juntos repetidamente em vários lugares do código. Isso indica que esses dados provavelmente pertencem a uma mesma entidade e deveriam ser encapsulados em uma classe ou estrutura.

Consequências do Data Clumps

  • Dificulta a manutenção e evolução do código.
  • Aumenta a chance de inconsistências e erros.
  • Viola o princípio de coesão e encapsulamento.
  • Prejudica a reutilização de código.
  • Pode levar à duplicação de lógica.
  • Dificulta a realização de testes automatizados.

Exemplo de Data Clumps e Refatoração

Antes (com Data Clumps)


// Exemplo em Java

void printClient(String name, String street, String city, String zip) {
    System.out.println(name + " - " + street + ", " + city + " - " + zip);
}

Depois (refatorado)


// Exemplo em Java

class Address {
    String street, city, zip;
    // construtor e métodos
}

void printClient(String name, Address address) {
    System.out.println(name + " - " + address.street + ", " + address.city + " - " + address.zip);
}
